# Configuration Reference

Complete reference for all Axon configuration files and environment variables.


---

## Server (`axon-server`)

### Quick Setup with `init`

The recommended way to configure a server is `axon-server init`:

```bash
axon-server init --listen :9090
```

This generates `~/.axon-server/config.yaml` with a random JWT secret, signs a no-expiry admin token, creates an initial join token, and initializes the SQLite database. See [Quick Start](quickstart.md) for the full flow.

### Config File

Path: passed via `--config` flag (default: `./config.yaml`)

### Full Example

```yaml
listen: ":9090"

tls:
  auto: false                             # TLS disabled by default; set true to auto-generate certs
  dir: "/var/lib/axon-server/tls"         # directory for auto-generated certs (default: ~/.axon-server/tls)
  cert: ""                                # path to explicit TLS certificate (disables auto-TLS)
  key: ""                                 # path to explicit TLS private key

auth:
  jwt_signing_key: "${AXON_JWT_SECRET}"   # HMAC-SHA256 signing key (required; auto-generated by init)

data:
  db_path: "/var/lib/axon-server/axon.db" # SQLite path for persistent data (default: in-memory)

heartbeat:
  interval: "10s"                         # heartbeat interval sent to agents (default: 10s)
  timeout: "30s"                          # mark offline after no heartbeat (default: 30s)

audit:
  db_path: "/var/lib/axon-server/audit.db"  # SQLite path for audit log (default: in-memory)
```

### Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `listen` | string | `:9090` | gRPC listen address |
| `tls.auto` | bool | `false` | Auto-generate self-signed CA + server certificate |
| `tls.dir` | string | `~/.axon-server/tls` | Directory for auto-generated TLS files |
| `tls.cert` | string | — | Path to TLS certificate (PEM). Overrides auto-TLS |
| `tls.key` | string | — | Path to TLS private key (PEM) |
| `auth.jwt_signing_key` | string | — | **Required.** HMAC-SHA256 key for JWT signing |
| `data.db_path` | string | `:memory:` | SQLite path for node registry, tokens, users, and join tokens |
| `heartbeat.interval` | duration | `10s` | Heartbeat interval for agents |
| `heartbeat.timeout` | duration | `30s` | Offline threshold |
| `audit.db_path` | string | `:memory:` | SQLite path for audit log (separate from data DB) |

### Environment Variables

| Variable | Overrides |
|----------|-----------|
| `AXON_JWT_SECRET` | `auth.jwt_signing_key` |
| `AXON_TLS_CERT` | `tls.cert` |
| `AXON_TLS_KEY` | `tls.key` |
| `AXON_TLS_DIR` | `tls.dir` |

### TLS

TLS is **disabled by default** — plaintext gRPC is suitable for internal/private networks.

To enable auto-TLS, either:
- Run `axon-server init --tls`
- Set `tls.auto: true` in the config file
- Provide explicit `tls.cert` + `tls.key`

#### Auto-TLS Details

When `tls.auto: true` and no `tls.cert`/`tls.key` is provided:

1. Server checks `tls.dir` for existing `ca.crt`
2. If missing: generates ECDSA P-256 CA (10-year validity) + server cert (1-year validity)
3. If `ca.crt` exists but `server.crt` is missing or expires within 30 days: regenerates server cert
4. CA fingerprint is logged for distribution to agents and CLI clients
5. During `axon-agent join`, the CA cert PEM is automatically sent to the agent

Generated files:

```
~/.axon-server/tls/
├── ca.crt          # CA certificate (auto-sent during agent join)
├── ca.key          # CA private key (keep secure, 0600)
├── server.crt      # Server certificate
└── server.key      # Server private key (0600)
```

### Data Persistence

All persistent data (node registry, tokens, join tokens) is stored in a single SQLite database with WAL mode. The path is set via `data.db_path`:

- `axon-server init` sets this to `~/.axon-server/axon.db` automatically
- If not set, defaults to in-memory (data lost on restart)
- Audit logs use a separate SQLite file (`audit.db_path`)

## Agent (`axon-agent`)

Config file path: `~/.axon-agent/config.yaml` (auto-created by `axon-agent join`)

### Enrollment

The recommended way to configure an agent is `axon-agent join`:

```bash
axon-agent join <server-addr> <join-token> --tls-insecure
```

This validates the token, receives an agent JWT, saves the config, and starts the agent. See [Quick Start](quickstart.md).

### Full Example

```yaml
server: "axon.example.com:9090"
token: "eyJhbGciOiJIUzI1NiIs..."  # agent JWT (issued during join)
node_id: "a1b2c3d4-..."            # assigned by server during join
node_name: "web-1"                  # user-specified or hostname
labels:
  env: production
  role: web
ca_cert: "/path/to/ca.crt"         # CA certificate for TLS verification (auto-saved during join with TLS)
```

### Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `server` | string | — | **Required.** Server gRPC address (`host:port`) |
| `token` | string | — | **Required.** Agent JWT for authentication |
| `node_id` | string | — | Auto-assigned by server during join |
| `node_name` | string | hostname | Human-readable node name |
| `labels` | map | — | Key-value labels for grouping |
| `ca_cert` | string | — | Path to CA certificate for TLS verification |

### Environment Variables

| Variable | Overrides |
|----------|-----------|
| `AXON_SERVER` | `server` |
| `AXON_TOKEN` | `token` |
| `AXON_CA_CERT` | `ca_cert` |

### TLS Behavior

The agent selects transport based on config:

1. `ca_cert` set → verify server cert against specified CA
2. Neither → plaintext gRPC (no TLS)

For auto-TLS setups, `axon-agent join` automatically saves the server's CA cert to `~/.axon-agent/ca.crt` and configures the agent to use it.

## CLI (`axon`)

Config file path: `~/.axon/config.yaml`

### Full Example

```yaml
server_addr: "axon.example.com:9090"
token: "eyJhbGciOiJIUzI1NiIs..."   # set by `axon config set token`
output_format: "table"               # "table" or "json"
ca_cert: "/path/to/ca.crt"          # CA certificate for TLS verification
```

### Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `server_addr` | string | — | Server gRPC address |
| `token` | string | — | JWT token (set by `axon config set token`) |
| `output_format` | string | `table` | Default output format |
| `ca_cert` | string | — | Path to CA certificate for TLS verification |

### Environment Variables

| Variable | Overrides |
|----------|-----------|
| `AXON_SERVER` | `server_addr` |
| `AXON_TOKEN` | `token` |
| `AXON_CA_CERT` | `ca_cert` |

### Global Flags

These flags override config-file values for any command:

```bash
axon --ca-cert /path/to/ca.crt <command>
```

### Config Commands

```bash
axon config set server axon.example.com:9090
axon config get server
```

Supported keys: `server`, `token`, `output_format`
